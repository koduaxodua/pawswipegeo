
## პრობლემა

მომხმარებელი ამბობს რომ "გამოტოვებული ძაღლების" ფუნქციონალი არ მუშაობს. კოდის შემოწმებისას ვხედავ რამდენიმე ლოგიკურ პრობლემას:

### 1. გამოტოვებულები არ ჩნდება — basic flow მუშაობს, მაგრამ:
`useLikedDogs.ts`-ში `dislikeDog` და `likeDog` ფუნქციები სწორად მუშაობს — left swipe → dislikedDogs-ში მიდის. ეს დადასტურდა ადრე end-to-end ტესტში.

### 2. რეალური ლოგიკური პრობლემები:

**a) Swipe stack-ში უკვე ნანახი ძაღლები ისევ ჩნდება**
`Index.tsx`-ში `dogs` მასივი მოიცავს ყველა ძაღლს, მათ შორის უკვე მოწონებულებს და გამოტოვებულებს. სვაიპის შემდეგ მხოლოდ index იზრდება, ფილტრაცია არ ხდება. თუ მომხმარებელი refresh-ს გააკეთებს, currentIndex ნულდება და უკვე ნანახი ძაღლები თავიდან გამოჩნდება — ამიტომ შთაბეჭდილება იქმნება რომ "გამოტოვებული არ ინახება".

**b) currentIndex არ ინახება localStorage-ში**
Refresh-ის შემდეგ index რესეტდება, რაც კიდევ უფრო აძლიერებს პრობლემას.

**c) "თავიდან დაწყება" ღილაკი არ ასუფთავებს history-ს**
უბრალოდ index-ს ანულებს, ანუ თუ უკვე ნანახია ყველა და reset დაიჭირე — იგივე ძაღლები გამოვა და ისევ "გამოტოვებულებში" ან "მოწონებულებში" იქნება, ანუ swipe არაფერს არ შეცვლის visually.

**d) როცა "გამოტოვებულიდან" → "მოწონებულში" გადააქვს (RotateCcw), ეს მუშაობს, მაგრამ პირიქით არ არსებობს** — მოწონებულიდან გამოტოვებულში გადატანის ღილაკი არ არის.

## გეგმა

### 1. გავფილტრო swipe stack უკვე ნანახი ძაღლებისგან
`src/pages/Index.tsx`-ში `dogs`-ის ნაცვლად გამოვიყენო ფილტრირებული მასივი:
```
availableDogs = dogs.filter(d => 
  !likedDogs.some(l => l.id === d.id) && 
  !dislikedDogs.some(s => s.id === d.id)
)
```
ასე უკვე მოწონებული/გამოტოვებული ძაღლები აღარ გამოჩნდება სვაიპში — ნანახი იქნება მხოლოდ ახლები.

### 2. currentIndex-ის გამარტივება
რადგან availableDogs თავად ფილტრავს ნანახებს, currentIndex ყოველთვის 0-დან იწყება. სვაიპის შემდეგ ძაღლი გადადის liked/disliked-ში და automatically ქრება availableDogs-დან, ანუ შემდეგი ავტომატურად ხდება პირველი. ეს უფრო მარტივი და სანდოა refresh-ისადმი.

### 3. "თავიდან დაწყება" ღილაკის ფუნქცია
ვინაიდან ფილტრაცია ავტომატურია, "თავიდან დაწყება" უნდა გაასუფთაოს ორივე სია (liked + disliked). ეს დავამატო explicit confirmation-ით: მომხმარებელს უნდა ესმოდეს რომ ეს არის reset.
ალტერნატივა: მხოლოდ disliked-ის გასუფთავება (რომ ხელახლა ნახოს გამოტოვებული, მოწონებულები შეინარჩუნოს). 

**არჩეული მიდგომა**: "თავიდან დაწყება" ღილაკი გაასუფთავებს მხოლოდ გამოტოვებულებს — ანუ მომხმარებელი ისევ ნახავს იმ ძაღლებს ვინც გამოტოვა, მაგრამ მოწონებულები შენარჩუნდება. ეს უფრო ლოგიკურია (მოწონებულები საჭიროა შესარჩევად).

### 4. Favorites-ში ორმხრივი გადატანა
`src/pages/Favorites.tsx`-ში:
- **მოწონებულების ტაბში** დავამატო ღილაკი რომ ძაღლი გადავიდეს გამოტოვებულებში (X icon).
- **გამოტოვებულების ტაბში** უკვე არის RotateCcw რომ მოწონებულებში გადავიდეს — ეს დავტოვო.
ამისთვის საჭიროა `useLikedDogs`-ში `likeDog`-ი უკვე ააშორებს disliked-დან (ეს უკვე ხდება), და დავამატო რომ `dislikeDog`-მა ააშოროს liked-დან (ეს უკვე ხდება). ანუ hook-ში არაფერი იცვლება, მხოლოდ UI-ში დავამატო ღილაკი.

### 5. ცარიელი სტეკის შეტყობინება უფრო ინფორმატიული
როცა availableDogs.length === 0, ვაჩვენო:
- რამდენი დარჩა მოწონებული/გამოტოვებული
- ღილაკი "გამოტოვებულების დაბრუნება" (resets dislikedDogs)

## შესაცვლელი ფაილები

1. **src/pages/Index.tsx** — დავამატო ფილტრაცია `useLikedDogs`-დან, შევცვალო `handleReset` ლოგიკა, გავამარტივო currentIndex (ან საერთოდ ამოვიღო — availableDogs[0] ყოველთვის შემდეგი ძაღლია).
2. **src/pages/Favorites.tsx** — მოწონებულების ტაბში დავამატო "გადატანა გამოტოვებულებში" ღილაკი (X icon, secondary action).

## ტექნიკური დეტალები

- Hook-ში ცვლილება არ საჭიროა — `likeDog`/`dislikeDog` უკვე სწორად ცვლის ორივე სიას.
- `currentIndex` state სრულად ამოიღება — ყოველთვის სვაიპავს `availableDogs[0]`-ს. ეს ბუნებრივია, რადგან სვაიპის შემდეგ ძაღლი ქრება ფილტრიდან.
- AnimatePresence-ში key არის dog.id, ამიტომ animation სწორად იმუშავებს როცა ძაღლი ქრება.
- "თავიდან დაწყება" → ახალი ფუნქცია `resetDisliked` `useLikedDogs`-ში (ან inline `setDislikedDogs([])`).
